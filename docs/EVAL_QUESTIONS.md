## 데이터 관련
- 왜 이 데이터를 선택했는가?
기업이 요구하는 기술, 우대 사항, 직무소개 등이 포함되어 있어 사용자의 역량과 비교하고 부족한 점을 분석하는데 적합한 데이터기 때문 
- 데이터의 어떤 부분이 신뢰할 수 없을 수 있는가?
기업마다 같은 직무라도 이름을 다르게 표현 가능함
- SQLite와 ChromaDB에 같은 데이터를 저장하는 이유는 무엇인가?
SQLite는 공고 정보를 구조화해 저장하고 관리하는 데이터베이스고, ChromaDB는 공고를 벡터로 저장해 사용자의 정보와 적합한 공고를 검색하는 RAG용 데이터베이스이기 때문
- 결측치와 중복을 어떻게 처리했는가?
결측치와 중복데이터는 제거한 후, SQLite와 ChromaDB에 저장

## 보안 관련
- .env 파일을 GitHub에 올리면 어떤 일이 생기는가? 
.env에는 API KEY같은 정보가 저장되기 때문에 이를 Github에 올리면 타인이 무단으로 사용하는 등의 보안 문제가 발생할 수 있다
- .env와 .env.example의 차이는 무엇인가? 
.env는 실제 API KEY와 환경 변수를 저장하는 파일이고 .env.example은 어떤 환경 변수가 필요한지 알려주는 예시 파일이다.

## Docker 관련
- Docker를 사용하는 이유는 무엇인가? 
어느 컴퓨터에서 실행해도 동일한 환경올 보장하기 위해 Docker 사용
- Dockerfile에서 EXPOSE와 CMD의 역할은 무엇인가?
EXPOSE는 컨테이너가 사용할 포트를 알려주는 역할, CMD는 컨테이너가 시작될 때 자동으로 실행할 명령 지정
